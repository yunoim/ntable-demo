>
> **E2E 플로우 테스트**: 이 세션은 헤드리스라 수동 브라우저 E2E 는 실행하지 못함. 시각 회귀·실사 QA 는 사용자 쪽에서 실행 필요.
>
> **R2 Notion 로그**: 이 세션에선 Notion MCP 도구를 로드하지 않았음. Notion 로그 DB 기록은 **deferred** — 사용자가 이어서 기록하거나, 추후 Notion 도구 로드 후 이 changelog 기반으로 업서트 필요.

- `public/guest.html` · `#profileEditModal` 상단에 이모지 아바타 + 「변경」 버튼 row 삽입. 기존 `openEmojiPicker()` (guest.html:3287) 재사용 — 공용 모듈로 추출하지 않음 (현재 1 callsite 확장).

### T-B0-2 · myplay(playlist-share) 작대기 단계 완전 제거
- `routes/question-sources.js` · `PACK_FLOW_DEFAULTS['playlist-share']` 에서 `'match'` 제거 → `['mvp', 'explore-result']`. 호스트의 `stepsToPhases()` 가 자동으로 match phase 를 제외.
- `PACK_DEFAULTS['playlist-share']` · `match_pairs_enabled: false`, `best_match_enabled: false`, `result_sections` 에서 `best_match·mutual_pairs` 제거. `labels` 에 `mvp_card_head · mvp_vote_card_head · closing_stage_sub` 추가.
- `routes/rooms.js` · `GET /rooms/:code` 응답 조립 시 `pack_id === 'playlist-share'` 인 방의 `closing_steps` 에서 `'match'` 를 read-time 필터 (기존 방 마이그레이션 대체).
- `routes/survey.js` · `POST /connections` 에 팩 정책 가드 추가 — `match_pairs_enabled === false` 면 `400 NOT_AVAILABLE_FOR_PACK` 반환 (프런트 우회 차단).
- `public/host.html` · `renderClosingSubsteps()` 가 `state.closingSteps` 기반으로 chip `hidden` + 번호 재부여. playlist-share 는 `isPlaylist` 분기에서 MVP chip 라벨을 「베스트 플리 투표/발표」 로 override. `stage-sub` 텍스트도 동적화 (`#closingStageSub`).

### T-B0-3 · 결과 페이지 상단 카드 → 「🎵 오늘의 베스트 플리」
- `public/result.html` · 페이지 2 `.card-title` 을 `#page2-card-title` 으로 식별. `data.pack_defaults.labels.page2_card_title` 있으면 동적 교체.
- playlist-share 전용 분기 추가 — `isPlaylistPack` 이면 MVP 1위 + `playlist_url` CTA 카드 렌더 (`else if (isCouplesPack)` 앞에 삽입). 도메인별 서비스 라벨(Spotify · Apple Music · YouTube Music · YouTube) 자동 매핑.
- `refreshConnectionMutuals()` 를 `window._packMatchPairsEnabled === false` 시 no-op. `applyPackGates()` 에서 플래그 세팅.
- `routes/survey.js` · `GET /api/result` 응답에 `mvp: {uuid, nickname, emoji, fi_count, playlist_url}` 필드 추가. playlist-share 일 때만 `playlist_links` 테이블에서 MVP 의 URL 조회.
- 「모임 종료」 버튼 조치: 지시서의 「삭제」 요구는 분석 결과 해당 없음 — 호스트의 closing 단계 마지막 phase 버튼 라벨은 `advanceClosingPhase()` 가 자동으로 "모임 종료" 로 붙이는 동작. match 단계가 빠지면 `closing-explore-result` 가 마지막이 되어 여전히 "모임 종료" 를 표시하는 건 의도된 동작 (호스트 전용, 게스트 `result.html` 에는 없음).

### T-B0-4 · 인스타 교환 게이트 — 프로필 값 자동 승계 + 「저장 중…」 stuck 해결
- `public/insta-exchange.html` · 저장 버튼 핸들러를 `try/catch/finally` 로 리팩터링. `AbortSignal` + `setTimeout(8000)` 타임아웃 가드. finally 에서 항상 `disabled=false; textContent='저장'` 리셋. 성공 시 `PUT /api/profile` 로 역동기화 (다음 모임 재사용).
- `public/guest.html` · `saveProfileEdit()` 에서 instagram 저장 성공 시 `POST /api/rooms/:code/me/instagram` 도 호출 → users 테이블과 room_members 양쪽 동기화.
- `routes/rooms.js` · `GET /rooms/:code/insta-status` 가 `room_members.instagram` 비어있으면 `users.instagram` fallback + 자동 upsert. 레거시 데이터에서도 게이트가 자동으로 뚫림.

### 비고 / 의도적으로 스킵한 것
- **`public/presenter.html` 별도 수정 없음**: presenter 의 match/mutual 섹션은 `STATE.currentPhase === 'closing-match*'` 에서만 렌더. host 가 해당 phase 를 트리거하지 않으면(=playlist-share 의 closing_steps 에 `'match'` 없음) 자연스럽게 skip. 별도 gate 불필요.
- **`public/shared/emoji-categories.js` 추출**: 지시서 명시 항목이나 단일 callsite 확장이라 YAGNI. 공용화 필요 시 후속.
- **「또 만나고 싶은 사람」 매칭 결과 카드 별도 삭제**: match_pairs_enabled 게이트 하나로 전체 섹션이 숨겨지므로 개별 DOM 삭제 불필요.
- **스크린샷 저장** (`docs/review/batch-0/`): 실사 QA 환경에서 before/after 촬영 필요 — 본 세션에선 생성 불가.
- **서버 응답 스키마 `{saved, handle}` 으로 변경하지 않음**: 기존 `{ok:true, instagram}` 를 유지. 클라이언트는 `r.ok && body.ok === true` 로 정합성 확인. 전체 엔드포인트 스키마 일원화는 별도 리팩터 과제로 분리 (insta 한 곳만 바꾸면 result.html 등 호출처도 함께 변경 필요).

- **T-17** · `login.html` 의 `scrollIntoView` 를 `getBoundingClientRect + scrollTo` 로 치환. iOS Safari + fixed header + safe-area 조합에서 레이아웃 jank 방지.

  - 본문·채팅·게스트 입력 이모지는 OS 기본 유지 (페이로드 절감)
