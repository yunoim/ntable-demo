# ntable 브랜드 자산

## 1. 브랜드 정체성

### 서비스 한 줄 정의
소셜 모임 퍼실리테이션 SaaS (오프라인·온라인·하이브리드). 모임장의 역할(아이스브레이킹·대화 유도·활동 진행·결과 정리)을 앱이 대신 수행해 누구나 모임장이 될 수 있게 만든다.

### 운영 모델
- Phase 1: 직영 쇼케이스 (남산엔테이블 기반 정기 모임)
- Phase 2: 모임장 모집 확산
- 솔로 운영 / 본업 겸직금지 / 파운더·실명·소속 노출 금지
- 법인 세무 정상화 진행 중 / 현재 매출 0원 / 노선 B (솔로 + 작은 혜택)

## 2. 공식 태그라인

### 메인
**"누구나 유잼 모임장이 되도록"**

### 핵심 포지셔닝 문구
"노잼 인간도 모임장이 되는 앱"

### 상황별 변형
- 짧은 버전: "유잼 모임장 메이커"
- 스토리형: "노잼 인간 ISTP도 모임장이 됩니다"
- 제품 설명형: "혼자선 어려웠던 모임, 앱과 함께하면 됩니다"

### 대표 스토리텔링 카피
```
저는 ISTP예요.
모임에서 입 떼기가 세상에서 제일 어려운.
그런 제가, 모임장이 됐습니다.
비밀은 이 앱에 있어요.
```

### 2-1. 공개 vs 내부 카피 분리 (2026-04-21 갱신)

| 용도 | 주 포지셔닝 | 적용처 |
| --- | --- | --- |
| **공개 카피** (대외 노출) | "누구나 모임장이 된다" · "모임 진행 도우미" · "QR 하나로 시작되는 모임 툴킷" | ntable.kr 랜딩, app.ntable.kr hero, 소셜 og·meta, 남산엔테이블 모집 퍼널 |
| **내부 스토리** (제한 노출) | "누구나 유잼 모임장이 되도록" · "노잼 인간도 모임장이 되는 앱" · ISTP 스토리 | 노션 허브, 세일즈·투자 자료, 창업 서사, 내부 블로그·스토리텔링 콘텐츠, 팀 온보딩 |

**왜 분리하는가**: "유잼·노잼" 일상어는 내부에서는 포지셔닝의 힘이 있지만, 이벤트 모집 퍼널에서는 (1) 자학 뉘앙스 부담 (2) 장난스러운 톤이라 서비스 신뢰도 저하 우려로 공개 카피에서 제외. 공개 카피는 "기능적·초대적" 톤으로 통일.

**메인 태그라인** (서비스 시그니처): 공개 = "누구나 모임장이 된다" (v1.2.0 — 단언형 전환) · 내부 = "누구나 유잼 모임장이 되도록". 서로 번역 관계지만 각자 채널에서만 사용. 이전 공개 카피 "누구나 모임장이 되는 모임"은 `public_main_legacy` 키로 보존 — 신규 노출 금지, 전면 교체 대상.

**위반 감지 시**: 공개 채널에 "유잼·노잼" 표현이 노출된 것을 발견하면 공개 카피로 즉시 교체. 내부 채널(노션·세일즈) 에서 "유잼" 이 희미해지면 스토리 위력 희석 — 내부 서사로 복원.

## 3. 표기 규칙

- 브랜드명 표기: **ntable** (소문자 고정)
- "유잼" 표기 고정 (유재 / YOUJAM / U-JAM 등 변형 금지)
- 시리즈 표기: "ntable Plays" · "ntable Games" · "ntable Draws" · "ntable Pitches" · "ntable Nights" (ntable 소문자 + 시리즈명 대문자 시작)
- 실명·법인명·소속·"주식회사" 등 절대 노출 금지
- 다른 브랜드명 혼동 표기 금지

## 4. 톤앤매너

### 전반 톤
- 친근하고 캐주얼
- 사용자를 "모임장"으로 격려
- 감성적·초대하는 말투 ("~해요", "~보내요")
- "유잼"·"노잼" 같은 일상어 적극 활용
- 이모지 적극 활용 (🍷✍️🎨📅⏰📍💰👥🎧🎁🏆)

### 기능 설명 원칙
기능 스펙보다 "그 기능으로 당신이 어떻게 되는지" 강조.

| Before | After |
| --- | --- |
| 이벤트를 생성하세요 | 당신의 첫 유잼 모임, 여기서 시작 |
| 체크인 완료 | 오늘의 유잼 현장에 도착! |
| 투표 제출 | 유잼 픽 한 표 완료 |

### 금지 표현
- "베타 테스트" / "실험" / "시범 운영" 등 미완성 뉘앙스
- 과도한 존댓말·권위적 어조
- 스타트업 클리셰 ("혁신", "플랫폼", "솔루션" 남발)
- 영어 남용 (한국어로 충분한 경우)

## 5. 시리즈별 컨텐츠 구조

| 시리즈 | 카테고리 | 참가비 (Phase 2) | 인원 | 심사 방식 |
| --- | --- | --- | --- | --- |
| ntable Nights | 대화형 | 30,000 | 8명 | 없음 or 참가선물 |
| ntable Plays | 음악 대회 | 35,000 | 8명 | 인기투표 (BEST + HIDDEN GEM) |
| ntable Games | 운동 대회 | 40,000 | 8명 | 기록순 랭킹 |
| ntable Draws | 창작 대회 | 35,000 | 8명 | 인기투표 |
| ntable Pitches | 비즈니스 대회 | 40~50,000 | 6~8명 | 심사 (항목별 점수) |

> 참가비 적용은 Phase 2 유료 전환 후. 현시점 앱·랜딩 카피에는 미노출.

## 8. UI 마이크로카피 (2026-04-22 신설)

게스트 온보딩·진행 중 안내·성공 피드백 같은 마이크로카피는 `brand.json` 의 `ui_copy` / `error_copy` 블록으로 분리해 관리. 이모지 프리픽스 시맨틱 고정: `✓` 성공 · `⚠️` 경고 · `✨` 초대 · `🎲` 액션(랜덤/재시도). 장식 남발 금지.

### 대표 사례 (Batch 1+2 도입)

| 상황 | 카피 | brand.json 키 |
| --- | --- | --- |
| 프로필 lazy prompt 카드 제목 | "✨ 프로필 더 채우기" + "2분" | `ui_copy.profile_lazy_prompt_title` · `profile_lazy_prompt_duration` |
| 프로필 저장 성공 토스트 | "✓ 프로필이 저장됐어요" | `ui_copy.profile_saved_success` |
| 닉네임 중복 실시간 경고 (inline) | "⚠️ 이 모임에서 '{nick}' 은 이미 사용 중이에요" | `error_copy.nickname_taken_inline` |
| 닉네임 이어받기 CTA | "이전에 이 닉으로 참여했다면 이어받기" | `ui_copy.nick_resume_cta` |

### v1.2.0 확장 (Batch 4 준비, 2026-04-22)

런타임 로더(`public/js/brand.js`) + 전면 카피 치환(T-18~T-23) 을 대비해 5개 블록 신설.

| 블록 | 키 수 | 용도 | 정책 주석 |
| --- | --- | --- | --- |
| `buttons` | 9 | 앱 내부 action 버튼 (cta.* 는 랜딩 전용) | `_buttons_policy` — 결과지향 동사만, "확인/다음/제출" 금지 |
| `wizard` | 7 필드 × 3 키 | 게스트 위저드 각 필드 q / helper / placeholder | `_wizard_policy` — q 는 대화체 질문, 명사 라벨 금지 |
| `empty_states` | 5 | 참가자·채팅·투표·매칭·대기 빈 상태 카피 | — |
| `toasts` | 8 | 환영·입장·저장·WS 재연결 등 단발 피드백 | `_toasts_policy` — `{nick}` 등 중괄호 템플릿 |
| `series.*.examples` | 5종 × 3 | 각 시리즈별 실제 모임 예시 (create.html 카드) | — |

**용어 금지 확장**: `tone.avoid` 에 `"똑똑한"`, `"AI가 알아서"` 추가. ntable 은 "도구" 포지셔닝이라 "AI 가 다 해줌" 뉘앙스 배제.

**런타임 로더 도입**: `usage_notes[1]` 을 "런타임 로드용이 아니라 참조 문서" → "런타임 로드용 (`public/js/brand.js` 가 fetch). 하드코딩 대신 `t()` 로 참조" 로 변경 예정 (T-18 에서 구현).

### 용어 통일

사용자 대면 텍스트에서 세션/공간을 가리킬 때는 **"모임"** 을 사용. "방"(room) 은 코드·DB 스키마 내부에서만 사용 (변수명 `room_code` 등). 톤을 따뜻하게 유지하기 위함.
